# 故事圣经 Schema(story-bible)

故事圣经是 novel-forge 的 keystone artifact。一本书一份 `bible.md`,放书目录根(cwd 模型,见 `storage-contract.md`)。

**核心原则**:圣经不是随手笔记,是**连续性漂移的总防线**。AI 长篇头号死因是跨章漂移(人物说话变样、设定前后矛盾、伏笔埋了不收)。每一块都因"杀掉一种具体漂移"才入选。每章定稿后由 `bible-updater` 从正文重新析出回灌(见 `bible-updater` agent),矛盾不静默覆盖、升级作者裁决。

## 8 块结构

### 1. 定位(positioning)
rubric 闭环的源头。立项(novel-kickoff)时设,起草与评审共用同一套权重。
- `爽文档位`:纯爽 / 偏爽 / **均衡(默认,金庸·基督山锚点)** / 偏文 / 纯文
- `流派`:玄幻 / 仙侠 / 都市 / 科幻 / 言情 / 历史 / …
- `平台`:起点男频 / 番茄 / 晋江女频 / 其他
- `日更字数`:目标单章字数(如 3000)
- `导出权重表`:由上述三维度查 `rubric.md` 的定位→权重表得出的每轴权重快照
> 防的漂移:写着写着偏离最初定位(爽文写成致郁文)。

### 2. 人物谱(characters)
每个有名角色一条:
- `姓名` / `身份` / `外在目标`(想要什么)/ `内在动机`(为什么)
- `声音特征`:说话方式、口头禅、用词习惯(**起草和连续性镜头都读这条**)
- `关系网`:与其他角色的关系
- `出场章` / `当前状态`(死活、所在、境界)/ `弧光轨迹`(性格变化路径)
> 防的漂移:人物声音漂移(同一角色不同章说话像两个人)。

### 3. 世界观设定(worldbuilding)
- `力量体系`:境界阶梯 / 修炼/能力规则
- `地理势力`:地图、门派、国家
- `规则`:世界运行的硬规则(什么能做什么不能)
- `术语表`:专有名词 ↔ 定义(**防止同一概念前后用词不一或定义漂移**)
> 防的漂移:设定前后矛盾(第 3 章说筑基不能飞,第 80 章筑基飞了)。

### 4. 金手指(golden-finger)
主角的核心优势。
- `是什么` / `机制与限制`(关键:限制,否则失控)/ `升级路径` / `当前等级`
> 防的漂移:金手指失控(越写越万能,张力崩)或忘了自己设的限制。

### 5. 伏笔账本(foreshadow-ledger) ★
每条伏笔一行,直接喂结构桥轴的"草蛇灰线"评审:
- `编号` / `埋设章` / `内容` / `计划回收章` / `状态`(待收 / 已收)
> 防的漂移:草蛇灰线断线——埋了伏笔忘了收,或回收时与埋设矛盾。`pending_伏笔` 计数同步到 state.json。

### 6. 剧情态(plot-state)
- `主线进度`:当前主线推进到哪
- `未解冲突`:当前悬而未决的矛盾
- `各线索位置`:多线并行时每条线当前在哪
- `时间线`:故事内时间推进(防时序错乱)
> 防的漂移:跨章剧情错乱、时间线打架。

### 7. 爽点节拍账(payoff-cadence-ledger) ★
喂留存轴评审:
- `已用爽点`:类型 + 出现章(复仇/打脸/抢婚/逆天/格局揭示…)
- `距上次爽点章距`:几章没有爽点了(防爽点荒)
- `章末钩子记录`:每章用的钩子类型(防钩子重复、防审美疲劳)
> 防的漂移:爽点荒(连续多章无爽点 → 流失)、钩子重复、审美疲劳。

### 8. 文风样本(voice-style-sample)
- `视角`:第几人称、限知/全知
- `叙述语气`:基调样例
- `锚定段落`:1-3 段范例正文,作为文笔锚点
> 防的漂移:跨章文风漂移(续写尤其依赖此块接住原作者笔法)。

## 一致性约束(crystal D-007)

- 每章定稿后 `bible-updater` 析出 diff 回灌;**矛盾告警 → 作者裁决 canon,不静默覆盖**。
- 续写/拆书由 `bible-reverse-extractor` 逆向批量生成,作者校验闸通过后才用。
- 评审的"连续性镜头"读**当前**圣经查本章一致性(与定稿后回灌分工不同)。
