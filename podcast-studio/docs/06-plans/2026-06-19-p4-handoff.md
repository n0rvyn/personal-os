## 3. 待办「小 bug」= committee 长度门（用户已定：P4 讨论 tradeoff 再实现）

- **现象**：committee 每稿 per-slice 长度门 `非空白字 ≥ 4500`（G2 gate），**任一稿不到线就整期 halt**。
- **根因**：MiniMax 草稿长度波动极大（run1: 3/3 过；加 Fix #2 后 run2: 拆步骤/B = 3931 不到线；再加提示词催长度 → **三稿全变短**(3661/2950/4271)→ 已回退）。**结论：靠提示词调长度不靠谱，别再试**（这是已证的死路，省得下个会话重踩）。
- **要做的是设计修法（三选一，跟当前架构权衡）**：
  - **(a) 只卡选中稿〔推荐〕**：长度门从 per-slice committee 挪到 `digest-select` 之后，只对最终要发的那一稿验 ≥4500。没选中的短稿直接丢、不该卡整期。改动小、最贴合「门管成品」。注意：scorer 本就倾向给短稿低分，所以选中稿通常不短。
  - **(b) 单稿有限重试**：哪稿不到线就重派那一稿（retry=1，复刻 `_RETRY_PARENT[faithfulness]` 的 retry-then-stop）。对波动最稳，代价多一次派发。
  - **(c) k-of-n 容忍**：3 稿 ≥2 过线即放行进选稿。改动小，但「选中那稿仍可能偏短」要靠评分尺挡。
- 触点：`lib/pipeline_papers.py`（committee step 的 gate）、`lib/runner.py`（per-slice gate 评估 `_run_agent_step` 约 1499 行附近 / 长度 gate `check_min_chars`-类）、`lib/paperline/`（如走 select-after 方案）。

## 5. 环境硬约束（下个会话必须先知道，否则重踩这次的 4 个坑）

1. **模型必须钉死 `--model sonnet`**：runner 的 `_default_dispatch` 不传 `--model`，CLI 默认会解析成 **Opus**（不是 MiniMax！）。跑 e2e 必须 monkeypatch `lib.dispatch.dispatch_persona` 注入 `model="sonnet"`。验证：`pgrep -fl "append-system-prompt" | grep -o "--model sonnet"`。见 memory `helper-subagents-use-sonnet`。
2. **arXiv 网络要 `no_proxy='*'`**：macOS 系统代理 `127.0.0.1:1082` 对 arXiv 时好时坏（503 / SSL-EOF）。urllib 默认走它。跑前 `export no_proxy='*' NO_PROXY='*'` 直连 arXiv。
3. **输出别放 `.claude/`**：`claude -p` 权限层拒绝写 `.claude/` 下的文件（committee 会静默写不出草稿）。sandbox 的 `output_dir` 必须在 `.claude/` 外。
4. **`--resume` 对论文线是坏的**：有个 station 每次 `make_scratch` 新目录，覆盖注入的 scratch_dir + resume glob → 无法续跑（部分跑挂了只能整跑）。是另一条已知 finding（非阻塞）。
