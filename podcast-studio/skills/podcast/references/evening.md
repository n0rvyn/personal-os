# /podcast evening — per-show editorial

This file is loaded by `skills/podcast/SKILL.md` when the user invokes
`/podcast evening`. It contains the per-step editorial instructions for the
persona subagents. It is **editorial prose, not orchestration logic**;
orchestration lives in `SKILL.md`.

## Show identity

- **Genre:** 晚间播客 (evening show).
- **Cadence:** daily, evening slot.
- **Length:** 目标约 7000 字（~20 分钟）；硬下限 6500 非空白字（≈18 分钟，产品下限），低于触发长度门 `check_min_chars` 扩写重跑.
- **Structure:** 3 段随笔中心 (essayistic, 脊柱反转；可证伪判断织入正文，不单列「我下注」格式段).
- **Voice:** 卞旸, 第一人称, 自然口语, 随笔不是评论.

## Per-step editorial (per the Adam evening template)

### Collection (达芬奇)

晚间播客主轴不是新闻——是 vault 里今天值得展开的一个张力/开放问题
/动摇中的信念。新闻、论文、资料降级成"为回答这个开放问题而检索来的
证据"。

1. 在 `vault.subjective_dir` 里定位今天真正值得展开的那个开放问题
   （一条具体的笔记，JSON 格式：`{id, title, excerpt, tension}`）。
   `tension` 字段要写成一句可独立读懂的话，例："X 和 Y 哪个先成熟"、
   "我以前说 Z，今天看可能反了"。
2. 围绕这个开放问题，构造 3-5 条证据线索 candidates——每条是
   `{domain: <tech|market|science|geo|culture 之一>, topic_tag: <英
   文短标签>}`，代表"我去哪个域、用哪个关键词检索证据来回应这个
   开放问题"。这些 candidates 是为回答开放问题服务的证据线索，不
   是当天主话题。
3. 用同一批 `--candidates` 和 `--pkos-note`，跑 3 次 `check`，每
   次加 `--show-type evening`（让 prep 把 `open_questions` 抬成
   brief 的主字段、把新闻候选降成次要 evidence）：

- 路 A: `--force-domain philosophy --force-contrarian lesswrong`
- 路 B: `--force-domain cognition --force-contrarian marginal-revolution`
- 路 C: `--force-domain history --force-contrarian stratechery`

3 条命令返回的完整 JSON brief 整段原样贴进素材摘要，分别放进 3 个
` ```json ` 代码块，代码块前用一行写明 `brief-A` / `brief-B` / `brief-C`。

### Drafts A/B/C (达芬奇) — 3 段随笔中心骨架

每路从素材摘要里取对应 `brief-X`，基于它写成播客文字稿（约 7000 字
~20 分钟），以卞旸第一人称写作，Persona=卞旸。每段是独立段落级
章节，段落之间用承接句（"顺着这条线往下看"/"另一件值得说的是"/
"如果把这事放到更大的坐标系里"/"说到这里我突然想到"）连接：

**叙事/情绪弧（Phase 5, 3 段结构之上的纵向情绪曲线）**：

晚间是随笔不是新闻。情绪曲线比早间内敛，但弧还是要立起来——
从开场一个具体的画面/未说出口的犹豫，经历一次视角上的轻轻
一推，最后落在一个"今晚我更倾向……但还有一件事没想清楚"
的诚实观察上。每段在 3 段结构之上承担弧上的具体情绪职责：

- **冷开场 (cold open)** —— ① 之前 30-60 字钩子：一句具体
  的画面/我最近一直没想明白的一个小事/一个反直觉的小观察；
  不要从"各位听众朋友"或"今晚我们来聊聊"起手。冷开场要
  直接把人扔进一个画面/一个未解的犹豫。
- **stakes（赌注被抬高）** —— ①-② 之间：把今晚那个开放
  问题抬到一个具体赌注上——"如果我倾向 A，那意味着 X；
  如果倾向 B，那意味着 Y"——用具体后果而非抽象意义抬 stakes。
- **turn（视角反转）** —— ②-③ 之间：用 ② 里的某个证据
  把同一件事换一个坐标系看——让听众意识到刚才 ①-② 的
  理解只是入口。
- **payoff + 自我披露（self-disclosure）** —— ③ 收尾段：本段
  收在一个"我之所以这么在意这件事，是因为……"的人格锚点
  上——让听众带走的不是知识点，而是"这人对这事是真的有
  感觉的"。本期若形成了一个可证伪判断，**就织进这个收尾观察里
  一起落**（"我赌 X 在 Y 时间窗内…，因为…"顺在话里说掉），不另
  起一个"我下注"格式段。

**vault-leads（用户的思考是主线，新闻/资料是镜头）**：

晚间档的核心论断应该来自 vault 里那条今天值得展开的张力/
开放问题/动摇中的信念，新闻、论文、资料降级成"为回答这个
开放问题而检索来的证据"。写法上 ① 段脊柱问题先于证据抬
出来——"我最近一直在想 X，理由是 Y，但今天读到一个证据
让我开始怀疑……"——把 vault 的长期观察先于新闻抬出来，
再用新闻去印证/动摇它。这一段不要写得像"参考了我之前
一篇笔记"，要用自然回味的口吻（"这半年我一直在想……"/
"前阵子我心里一直有一个不太说得清的事……"）。

1. **① 开口·抛一个张力/开放问题/动摇的信念**（写在最前面）—— Upstream
   Artifacts 注入了「最近开放问题段」（含**早间今天刚写下的注**，和
   本档以来尚未结算的开放判断）和「到期旧注结算段」。「最近开放
   问题段」**优先用**：先承接/呼应它（"我今早提的那个 X 还在
   转..."/"早间档那条没下完的 Y 顺着晚上想..."），再切到
   `brief-X.open_questions` 里的具体张力——不是抛新闻，是抛一个真实
   的疑问或纠结。`open_questions` 里有多条开放问题，挑一条最能撬
   动今晚思考的、挑它的角度要落在"我最近为什么在犹豫"上。
2. **② 围绕这个问题展开** —— 把 `brief-X.evidence` 里的新闻/论文/
   资料当作"为回答这个开放问题而检索来的证据"引进来；边想边谈，
   引一个证据就走一步思考，再引一个证据再走一步。证据是论证材
   料，不是主题。不要把它们包成"今日资讯速览"——是把它们和开
   放问题连成一条线。
3. **③ 落在一个有温度的观察上（整期收尾）** —— 不强行下大结论。
   结尾是一段诚实的观察：可以是一个暂时的想法（"到现在我更倾向
   于..."）、一个新的疑问（"但还有一件事我还没想清楚..."）、或一段
   诚实的犹豫（"这一类的事我还在摇摆..."）。这一段不是定论，是当晚
   的一个观察、一个停顿、一个呼吸。**本期若在 ①-③ 里形成了一个可
   证伪的判断**（具体时间窗口/路径/可观测信号），就**织进这段收尾观
   察里**自然落下（"我赌 X 在 Y 时间窗内不发生，因为…"），下游会从
   正文蒸馏它进台账——但**不另起一个"我下注"格式段/清单**。今天没
   有真东西可赌，就不赌：不硬凑、不把"暂时想法"包装成赌注。

**「上期成绩单」戏剧化结算段（Phase 5, dramatized settlement）**：

晚间档的 ① 段开场自我证伪结算不再只是平铺直叙的"上次我
说了 X，证据走向 Y"——把它做成一个戏剧化的小段，**用一句
固定的开场语和一句固定的收束语包住**，让它听起来像一个
**有标签的固定环节**（类似电台的"档案柜时间"），而不是
正文里随便一段。
   - 固定开场语：例如"上期成绩单" / "上次的话，回来对一下账"。
   - 固定收束语：例如"这一段到这里，下一件事——"。
   - 中间是真正的 1-2 个 due bet 结算（"上期我说 X，证据走到
     Y，所以我对/错在……"）。
   - 如果这一期没有 due bet，**整段就跳过**——不要为了对齐
     模板而硬写"今天没有要结算的"，那是凑字。

**硬约束**：

- 严禁伪造第一人称证据
- 命名仪式/亲验/反方对比不是每期必填的强制槽位——是"挣来才有"
- **严禁念出内部存储/工具/字段名**：稿子里绝不许出现 PKOS、
  GetNote、vault、笔记库、pkos_note、brief 这类内部把手名——
  它们只是你取数据用的把手，不是给听众听的。引用这些材料时
  只聊想法本身，出处用自然回味（"最近一直在想…"/"前两天读
  到个说法，一直搁在心里…"），绝不报账式溯源
- **这期是随笔不是评论**：不要把多个开放问题铺成清单（"第
  一...第二...第三..."）——只挑一条脊柱问题深挖
- brief 里 `pkos_note` 至少出现一次并与脊柱问题连接（引用带
  时间锚）；`cross_domain_candidates`（本路强制 domain 已在
  Collection 步指定）至少用一条做真实的跨域碰撞；`contrarian_source`
  做一次反方对比；`self_past_candidates` 有真实矛盾才写自我
  对话，否则跳过
- 严格按 `brief.open_questions` 选脊柱问题写稿，不允许写 brief
  之外的 topic

**输出**（晚间档的写稿步骤）：回复正文必须就是这篇播客文字稿
本身，从 ① 开口·抛开放问题到 ③ 落在一个有温度的观察（可证伪判断
织入此收尾），完整、约 7000 字（不得向下缩水），直接写在回复里。

## Morning → evening carry (Phase 3 hook — not wired here)

晚间档写稿的 ① 段优先承接**早间今天刚写下的注**和**本档以来尚
未结算的开放判断**。这个承接关系在 Phase 2 里是"占位说明"——`vault`
里目前**没有自动注入 Upstream 的开放问题段**，写稿步骤也不会真的
拿到「最近开放问题段」。

Phase 3 会把 stance card + belief ledger 接进来：

- 早间档定稿后，把本期产生的开放问题（来自写稿 ① 自我证伪段 +
  评分未完结的判断）写到 ledger。
- 晚间档采集时，从 ledger 拉「最近开放问题段」注入 Upstream Artifacts，
  让晚间档的 ① 段真的能承接。
- 晚间档定稿后，归还新一轮开放问题到 ledger。

本 phase 不实现该 wiring；该 hook 在本文件 + 早间档的 references
+ `SKILL.md` 里都是 Phase-3 占位说明。Phase 2 的晚间档是**没有自动
承接**的纯手写 evening editorial。
